# QuickPRD

AI 驱动的产品需求分析工具。从一句模糊需求到可评审的 PRD，全流程自动化。

## 工作流

```
模糊需求 → AI 生成澄清问题 → 填写答案 → 生成 PRD 初稿 → 技术/风控评审（MVP 视角） → 确认待决策问题 → 生成最终版 PRD → 生成业务流程图 / 页面结构说明 / 时序图（可选）
```

## 功能

### 需求分析

- **AI 智能分析** — 通过 DeepSeek 大模型深度理解需求语义，生成针对性的澄清问题（非模板化）
- **6 维度覆盖** — 从业务目标、用户范围、核心指标、关键规则、边界异常、风控/技术约束全面审视
- **需求类型模板** — 支持 4 种需求类型（通用 / 营销活动 / 电商交易 / 后台管理），选择后自动注入领域专属的分析维度、PRD 章节结构和评审侧重点
- **示例需求** — 提供 3 个预设示例，点击即可体验

### PRD 生成

- **PRD 初稿** — 基于需求和问答自动生成结构化 PRD 文档（背景与目标、功能说明、业务规则、边界与异常）
- **技术/风控评审（MVP 视角）** — AI 以技术负责人 + 风控负责人视角审查 PRD，所有修改建议均为最小可行方案，按严重程度输出问题和建议
- **待确认问题** — 评审后自动提取需要产品决策的关键问题，用户可选填回答，未回答的按最简方案处理
- **最终版 PRD** — 将评审建议和用户补充回答融入 PRD，生成修订后的最终文档
- **业务流程图** — AI 分析最终 PRD，自动识别需要图示化的关键流程，生成 Mermaid 流程图；若 PRD 较简单则提示无需流程图
- **页面结构说明** — AI 识别 PRD 中涉及的用户界面页面，输出低保真原型结构描述（布局区块、组件清单、页面状态）；若不涉及前端页面则提示无需生成
- **时序图** — AI 分析最终 PRD，识别多系统/多服务间的交互流程，生成 Mermaid 时序图；若不涉及系统间交互则提示无需生成

### 界面设计

- **清爽文档风** — 参考 Notion / 飞书的视觉风格，大留白、柔和圆角、轻阴影，沉稳蓝主色调
- **CSS 设计系统** — 基于 CSS 自定义属性的完整变量体系（色彩、间距、圆角、阴影、过渡），统一的按钮 / 输入框 / 卡片组件类
- **步骤指示器** — 顶部 5 步导航条（需求输入 → 需求澄清 → PRD 生成 → 评审优化 → 产出物），实时高亮当前步骤
- **模态框动画** — 弹窗开关带 fade + scale 过渡效果
- **骨架屏加载** — AI 分析时显示 shimmer 骨架屏，替代传统 spinner
- **Toast 通知** — 全局浮动通知（成功 / 警告 / 错误），替代原生 alert
- **工作流过渡动画** — 步骤切换带淡出/淡入过渡效果，视觉衔接流畅
- **流式进度条** — AI 生成期间页面顶部显示蓝色脉冲进度条，明确反馈处理状态
- **按钮脉冲态** — 异步操作期间按钮带呼吸动画，视觉提示生成进行中
- **键盘焦点样式** — Tab 键导航时显示蓝色焦点轮廓，提升键盘可访问性
- **减弱动效支持** — 尊重系统「减少动态效果」偏好设置
- **ARIA 无障碍** — 关键区域添加 `aria-live`、`aria-label`、`aria-current` 属性

### 体验优化

- **流式输出** — AI 响应逐步渲染，无需等待全部完成
- **会话自动持久化** — 每个步骤完成后自动保存全部状态，刷新页面恢复到上次所在阶段（PRD、评审、流程图等均不丢失）
- **新建会话** — 一键清空当前工作区，回到初始状态重新开始，不影响历史记录
- **历史记录** — 保存最近 20 条分析记录，随时回溯，清空前需二次确认
- **快捷键** — Ctrl+Enter / Cmd+Enter 快速提交；ESC 关闭弹窗
- **导出与复制** — 澄清文档、PRD、评审报告均支持 Markdown 导出和一键复制；复制失败时明确提示
- **终稿快速返回** — 从问题清单页面可一键返回最终 PRD 文档
- **Prompt 版本管理** — 支持多版本 Prompt 管理，可在设置面板切换
- **防重复提交** — 异步操作期间自动锁定按钮（带脉冲动画反馈），防止重复调用浪费 API 额度
- **输入保护** — 超长输入、空答案生成 PRD 时给出提示；重新生成时确认是否丢弃已有流程图/页面结构
- **输入框自动扩展** — 答案输入框随内容自动增高，无需手动拖拽

## 使用方式

### 1. 获取 API Key

支持两种 API 提供商：

- **DeepSeek（直连）**：前往 [platform.deepseek.com](https://platform.deepseek.com/api_keys) 获取
- **OpenRouter**：前往 [openrouter.ai](https://openrouter.ai/keys) 获取

首次打开工具时会自动弹出设置框，粘贴 Key 即可。

### 2. 启动工具

双击打开 `index.html` 即可使用，无需安装任何依赖。

也可以通过本地服务器启动：

```bash
npx http-server -p 8080
```

然后访问 `http://localhost:8080`。

### 3. 完整使用流程

1. 输入一段模糊的产品需求描述（或点击示例按钮），并选择需求类型（默认为通用需求）
2. 点击"生成澄清问题"，AI 按 6 个维度生成针对性问题（不同需求类型会补充领域专属的分析维度）
3. 在问题下方填写补充信息（可选）
4. 点击"生成 PRD 文档"，AI 基于需求和问答生成 PRD 初稿
5. 点击"风险评审"，AI 从技术和风控视角审查 PRD，给出 MVP 修改建议
6. 查看并回答评审提出的待确认问题（可选，留空则按最简方案处理）
7. 点击"按照修改建议进行补充并生成最终 PRD"，AI 融入评审意见和补充回答生成最终版
8. 点击"生成业务流程图"，AI 自动识别关键业务流程并生成可视化流程图（可选）
9. 点击"生成页面结构说明"，AI 输出涉及页面的低保真原型结构描述（可选）
10. 点击"生成时序图"，AI 识别多系统间交互流程并生成时序图（可选）
11. 导出或复制最终 PRD、流程图、页面结构说明和时序图用于评审

## 文件结构

```
├── index.html    # 页面结构、步骤指示器、模态框
├── style.css     # CSS 设计系统（变量 + 组件类）与全局样式
├── prompts.js    # Prompt 版本管理、PRD 模板库 & 全局常量（STORAGE_KEYS）
├── api.js        # AI API 调用层（流式 SSE + DeepSeek/OpenRouter 双通道）
├── export.js     # 文档导出（Markdown 生成、渲染与下载）
├── script.js     # UI 编排、交互逻辑与步骤状态管理
├── CLAUDE.md     # Claude Code 协作上下文
└── README.md
```

## 技术方案

- 纯 HTML / CSS / JS，零构建
- CSS 设计系统：基于 `:root` 自定义属性的变量体系，统一管理色彩、间距、圆角、阴影、过渡、字体阶梯；所有 font-size 和语义色通过变量引用
- 字体：Inter（Google Fonts CDN）+ PingFang SC + Microsoft YaHei
- 支持 DeepSeek 直连和 OpenRouter 两种 API 通道
- 全链路流式输出（SSE），实时渲染 AI 响应
- 会话状态全量持久化到 localStorage，刷新页面自动恢复到最近工作阶段；localStorage 写入异常时通过 Toast 提示用户清理空间
- API Key 仅存储在浏览器 localStorage，不上传任何服务器
- XSS 防护：所有动态内容经过 HTML 转义
- Mermaid.js v10（CDN）渲染业务流程图与时序图，顺序渲染避免 ID 冲突，支持渲染失败 fallback 到源码查看
- Clipboard 降级：兼容 file:// 协议下的复制操作，失败时给出明确反馈
- 模态框动画：fade + scale 过渡，transitionend 事件驱动 + setTimeout 兜底
- 可访问性：WCAG AA 色彩对比度、`:focus-visible` 焦点样式、`prefers-reduced-motion` 减弱动效、ARIA 语义标注
